## Overview

Page navigation allows you to navigate to standard Salesforce pages like the home page, Chatter, Files, and custom tabs. This is useful for creating custom navigation experiences in your Lightning Web Components.

## Navigate to Home Page

Navigate to the Salesforce home page using the `standard__namedPage` type.

<CodeGroup>
```javascript navToHome.js
import { LightningElement } from 'lwc';
import { NavigationMixin } from 'lightning/navigation';

export default class NavToHome extends NavigationMixin(LightningElement) {
    navigateToHome() {
        this[NavigationMixin.Navigate]({
            type: 'standard__namedPage',
            attributes: {
                pageName: 'home'
            }
        });
    }
}
```

```html navToHome.html
<template>
    <lightning-card title="NavToHome">
        <lightning-button
            label="Go to Home"
            onclick={navigateToHome}
        ></lightning-button>
    </lightning-card>
</template>
```
</CodeGroup>

## Navigate to Files Home

Navigate to the Files home page using the `standard__objectPage` type.

<CodeGroup>
```javascript navToFilesHome.js
import { LightningElement } from 'lwc';
import { NavigationMixin } from 'lightning/navigation';

export default class NavToFilesHome extends NavigationMixin(LightningElement) {
    navigateToFilesHome() {
        this[NavigationMixin.Navigate]({
            type: 'standard__objectPage',
            attributes: {
                objectApiName: 'ContentDocument',
                actionName: 'home'
            }
        });
    }
}
```

```html navToFilesHome.html
<template>
    <lightning-card title="NavToFilesHome">
        <lightning-button
            label="Go to Files"
            onclick={navigateToFilesHome}
        ></lightning-button>
    </lightning-card>
</template>
```
</CodeGroup>

## PageReference Types for Pages

### standard__namedPage

Navigate to standard Salesforce named pages.

<ParamField path="type" type="string" required>
  Must be `standard__namedPage`
</ParamField>

<ParamField path="attributes.pageName" type="string" required>
  The name of the page to navigate to
</ParamField>

**Common Page Names:**
- `home` - Salesforce home page
- `chatter` - Chatter home page

### standard__navItemPage

Navigate to custom tabs.

<ParamField path="type" type="string" required>
  Must be `standard__navItemPage`
</ParamField>

<ParamField path="attributes.apiName" type="string" required>
  The API name of the custom tab
</ParamField>

### standard__objectPage (for home)

Navigate to object home pages.

<ParamField path="type" type="string" required>
  Must be `standard__objectPage`
</ParamField>

<ParamField path="attributes.objectApiName" type="string" required>
  The API name of the object (e.g., 'ContentDocument' for Files)
</ParamField>

<ParamField path="attributes.actionName" type="string" required>
  Must be `home` for object home pages
</ParamField>

## Summary Table

| Destination | Type | Attributes |
|-------------|------|------------|
| Home Page | `standard__namedPage` | `pageName: 'home'` |
| Chatter | `standard__namedPage` | `pageName: 'chatter'` |
| Files | `standard__objectPage` | `objectApiName: 'ContentDocument'`, `actionName: 'home'` |
| Custom Tab | `standard__navItemPage` | `apiName: 'YourTabName'` |

## Headless Action Example

You can also use navigation in headless actions (components without UI that can be invoked from Quick Actions).

```javascript navigateToRecordHeadlessAction.js
import { LightningElement, api } from 'lwc';
import { NavigationMixin } from 'lightning/navigation';

export default class NavigateToRecordHeadlessAction extends NavigationMixin(
    LightningElement
) {
    @api invoke() {
        this[NavigationMixin.Navigate]({
            type: 'standard__objectPage',
            attributes: {
                objectApiName: 'Contact',
                actionName: 'home'
            }
        });
    }
}
```

**Source:** [navigateToRecordHeadlessAction.js:7](~/workspace/source/force-app/main/default/lwc/navigateToRecordHeadlessAction/navigateToRecordHeadlessAction.js)

<Tip>
Headless actions are useful for creating Quick Actions that perform navigation without displaying a UI.
</Tip>
